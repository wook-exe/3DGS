# 📋 AIOSS 프로젝트 칸반 워크플로우 가이드

본 프로젝트는 GitHub Projects 기반의 칸반(Kanban) 보드를 활용하여 이슈와 태스크를 추적합니다. 모든 작업 내역은 아래의 상태(Status) 흐름을 따라 관리됩니다.

## 📌 컬럼(Status) 정의 및 사용 규칙

### 1. 🗄️ Backlog (백로그)
*   **정의:** 앞으로 해야 할 모든 아이디어, 버그 리포트, 기획안이 모이는 보관소입니다.
*   **액션:** 아직 구체적인 작업 일정이 잡히지 않은 이슈를 이곳에 생성합니다.
*   *예시:* "HMD 기기(Meta Quest) 연동 가능성 조사", "렌더링 프레임 10% 개선 아이디어"

### 2. 📝 To Do (할 일)
*   **정의:** 이번 주(혹은 이번 스프린트)에 작업하기로 확정된 이슈입니다.
*   **액션:** Backlog에서 우선순위가 높은 이슈를 골라 To Do로 이동시킵니다. 작업 담당자(Assignee)가 반드시 지정되어야 합니다.
*   *예시:* "Colmap을 이용한 카메라 포즈 추출 스크립트 작성"

### 3. 🚧 In Progress (진행 중)
*   **정의:** 담당자가 현재 실제로 개발 또는 작업을 진행하고 있는 상태입니다.
*   **액션:** 로컬에서 브랜치를 생성(`feature/issue-number`)하고 코딩을 시작할 때, 카드를 이 컬럼으로 옮깁니다.

### 4. 👀 Review (리뷰 대기 및 진행 중)
*   **정의:** 작업이 완료되어 GitHub에 Pull Request(PR)가 생성된 상태입니다.
*   **액션:** PR을 생성할 때 관련된 이슈를 연결(Linked Issues)합니다. 동료의 코드 리뷰를 받고 수정 사항을 반영하는 단계입니다. 자동화 설정 시 PR이 열리면 카드가 자동으로 이 컬럼으로 이동합니다.

### 5. ✅ Done (완료)
*   **정의:** 코드 리뷰가 통과되고 `main` 브랜치에 성공적으로 Merge(병합)된 상태입니다.
*   **액션:** PR이 Merge되어 닫히면 연결된 이슈도 자동으로 닫히며 이 컬럼으로 이동합니다. (GitHub 내장 자동화 적용)

---

## 💡 GitHub 자동화(Workflow) 팁
GitHub Project의 자체 내장 **Workflows** 기능을 켜두면 관리가 편해집니다.
1. **Item added to project:** 이슈가 생성되면 자동으로 `Backlog`로 이동
2. **Pull request opened:** PR이 생성되면 연결된 이슈가 `Review`로 이동
3. **Pull request merged:** PR이 병합되면 연결된 이슈가 자동으로 `Done`으로 이동
